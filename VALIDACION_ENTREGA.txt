EDUFLOW — VALIDACIÓN DE ENTREGA FINAL

Base utilizada:
EduFlow final con notificaciones, sensor de proximidad, portadas Canvas y reproductor de audio.

API configurada:
https://testappkmp-production-e13d.up.railway.app

Cambios incluidos:
- Examen obligatorio al crear una materia y revisión mensual posterior.
- Bloqueo de tarjetas y audios sin examen mensual o durante el día del examen.
- Validación equivalente en interfaz y backend, incluido el archivo de audio.
- Recordatorios por dificultad y cercanía del examen.
- Selectores para dificultad, tipo/fecha de examen y grado académico.
- Logo vectorial propio de EduFlow.
- Avatares Canvas de animales para alumnos y avatar por grado para asesores.
- Uso del avatar real en encabezados, menú, perfil y Comunidad.
- Panel de asesor ampliado sin acceso a materiales privados de alumnos.
- Navegación compatible con el gesto Atrás de Android.
- Barra inferior completamente en español.
- Diálogos mejorados para crear tarjetas y audios.
- Fecha del backend ajustada a America/Mexico_City, configurable con APP_TIME_ZONE.
- Protección frente a falsos avisos de examen cuando falla temporalmente la red.

Revisión estática realizada:
- Archivos JSON y XML analizados correctamente.
- Kotlin revisado con kotlinc para detectar errores sintácticos; no se encontraron errores de parser.
- Contratos expect/actual presentes para Android, JS y Wasm.
- URL de ApiConfig verificada.
- Permisos INTERNET, POST_NOTIFICATIONS y característica TYPE_PROXIMITY verificados.
- Rutas de tarjetas y podcasts verificadas contra MaterialAccessPolicy.
- Ausencia de etiquetas visibles Dashboard, Peers y SF comprobada.
- Navegación inferior en español comprobada.

Limitación del entorno:
No fue posible ejecutar la compilación Gradle completa porque Gradle 9.1.0 debe descargarse desde services.gradle.org y este entorno bloquea la descarga del archivo ZIP. La versión debe sincronizarse en Android Studio, desplegar nuevamente el backend en Railway y probarse en un dispositivo Android.
